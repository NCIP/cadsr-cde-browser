An Overview of the CDE Browser

The Common Data Element (CDE) Browser is a web-based tool designed to let you search for, view, and export Data Elements (DEs). DEs are descriptors of data that is, metadata that are used to set up data collection forms for cancer research studies. The DEs available for viewing using the CDE Browser are registered and stored in the Cancer Data Standards Repository (caDSR). More information about the caDSR project can be found at http://ncicb.nci.nih.gov/core/caDSR.

The CDE Browser can also be used to view basic information related to forms, templates, standards, and other specialized collections of Data Elements. However, if you need to create or edit forms, we recommend that you use the Form Builder tool.

 

The CDE Browser opens to the Data Element Search page. This page contains the main Navigation Menu, the caDSR Search Tree, and the Basic Data Element Search Pane.

Searching for Data Elements (DEs)

There are several different ways to search for Data Elements. The best way to search will depend on what you are trying to find.

         Generally, if you are looking for a collection of DEs (for example, if you are trying to find the DEs used on an existing form or trying to find the DEs associated with a standard), you should use the caDSR Search Tree.

         Otherwise, your best bet is to search by keyword using either Basic Search or Advanced Search.

 

Regardless of which method you use to search for DEs, the results of your search will always appear in the Data Element Search Results table.

The Basics of Searching by Keyword

To enter a query into the CDE Browser, just type in the term you want to search on and hit the Enter key (or click the Search button).

Use the Radio Buttons to adjust how the Keyword Search is performed. See the section below on Keyword Search Options
 

Click the New Search button to initiate a new search. This clears any text youve entered above. It also clears any results displaying in the search results table.
 

 

 

 

 

The Browser searches for an exact match of the search term(s) in the Long Name, Preferred Name, Document Text, Definition, Reference Documents, and Designations. The diagram below shows some of the main fields associated with each Data Element. If you search for gene, the Browser will look for an exact match of gene in the fields highlighted below.

 

Capitalization

CDE Browser searches are not case sensitive. The Browser ignores capital letters when searching for Data Elements. Gene and gene return the same matches.

 

Using Wildcards to Broaden a Search

To broaden your search, use the wildcard character (*). The wildcard character can be used more than once within a search term.

Tips on Using Wildcards

         To search for a Data Element that contains the word gene somewhere in its name, definition, or in the title of attached reference documents, put a wildcard before and after the term.

         To search for Data Elements that begin with the word gene, put a wildcard after the term.

         To search for matching phrases, use wildcards to bridge words. For example, using gene*brother* will return a match for Genetic Full Brother Count and Genetic Half Brother Count.

 

Keyword Search Options

The radio buttons to the left of the keyword search item provide you with three options for conducting your keyword search:

  1. Exact Phrase: This is the search that was previously available the search uses key words and wildcard characters (*)
  2. All the Words: The search insures that all the words in the search box are present before returning a CDE to the hitlist (This is the same as performing a logical AND on the keywords.)
  3. Any of the Words: The search will return CDEs based on the presence of any individual keyword in the search box. (This is the same as performing a logical OR on the keywords.)

 

Searching for a DE by Public ID

 

Each Data Element in the caDSR has a unique Public ID. If you know the ID of the Data Element youre searching for, enter it into the search box and select Public ID from the drop down list. (Note: You can use wildcards when searching by Public ID.)

Using the Advanced Search Page

The Advanced Search page gives you greater control over the search parameters. You can opt to search by name, ID, concept, or by a number of other attributes. You can combine criteria to create complex searches.

 

Searching by Concept

Concepts were added to the caDSR model with version 3.0. Concepts were added to provide additional semantic support. As a result, searching by concept can be a powerful way to find the appropriate data elements.

 


 

 

 

 


The Browser will search object classes, properties, and value domains for matches. If a match is found, the Browser will return the data element associated with the object class, property, or value domain that matched the Concept Name or Code.

 

Searching by Attributes

CDEs are represented in the caDSR database using the ISO/IEC 11179 standard.  This standard is somewhat complex, but it offers a richly expressive model for metadata that does a good job of supporting the variations needed for biomedical applications. For more information on the use of ISO 11179 in caDSR, go to http://ncicb.nci.nih.gov/NCICB/core/caDSR/ISO11179.

The CDE Browser lets you search for data elements using the following ISO11179-based attributes:

  • Public ID: To search for data elements based on Public ID, enter a valid Public ID in the field provided. You can add an asterisk (*) wildcard character to broaden the search. The wildcard character can be used at the beginning, middle, or end of the search term.
  • Data Element Concept: The Data Element Concept is the component of the data element that indicates the semantic meaning of the data element. To filter data elements based on Data Element Concept, click on the icon and proceed as described above for Value Domain.
  • Object Class: Data Element Concepts contain at least one Object Class. Use this filter to find data elements that have Object Classes that match the keyword or phrase you enter here. Wildcards are supported.
  • Property: Data Element Concepts can contain a Property. Use this filter to find data elements that have a Property term that matches the keyword or phrase you enter here. Wildcards are supported.
  • Value Domain: The Value Domain is the component of the data element that indicates the permissible values that can be collected in an actual research study. To filter data elements based on Value Domain, click on the icon. A popup window will appear that allows you to do a keyword search for the Value Domain of interest. Once the results appear in the popup, click on the particular Value Domain you want to use in your data element search. The Value Domain you selected will automatically be entered into the search criteria box.
  • Permissible Value: To filter data elements based on their permissible values, enter a value in the field provided. You can add an asterisk (*) wildcard character to broaden the search. The wildcard character can be used at the beginning, middle, or end of the search term.
  • Classification: Classification Scheme Items are particular classifications that data elements can be assigned to by the Context administrator. To filter data elements based on their Classification Scheme Item assignment, click on the icon to search for and select the Classification Scheme Item of interest, similar to what is described above for Value Domain.
  • Alternate Name: A data element is allowed to have multiple alternate names in multiple contexts. This search criterion should be used for searching data elements based on alternate names. It is used in conjunction with alternate name type(s) search criteria.
  • Alternate Name Type(s): This list displays all the alternate name types. Multiple values can be selected from this list by holding the ctrl key. Alternate name type has to be specified for a search based on alternate name.

 

Search Result Filters

  • Version: Data Elements are assigned version numbers by the Context administrator. To filter data elements based on their version, select the Latest Version or All Versions radio button.
  • Workflow Status: The Workflow Status is the administrative status of the data element. In most cases you will probably want to search for data elements with a status of RELEASED in the name, but any status is available as a search criterion. To filter data elements based on their Workflow Status, use the multi-selection drop down list and select the Workflow Statuses of interest. Multiple workflow statuses can be selected from the drop down list by holding the Ctrl key.

         Registration Status: To filter data elements based on their Registration Status, use the multi-selection drop down list to select the Registration Statuses youre interested in. Multiple registration statuses can be selected from the drop down list by holding the Ctrl key.

         Context Use: A data element belongs to the owning Context that has administrative authority to edit and update it. In addition to belonging to an owning Context, a data element can be designated for use by another non-owning Context. In this version of CDE Browser, it is possible to search for data elements that are Used, Owned, or Owned By/Used By a particular Context. Owned By/Used By is the default setting.

Searching for DEs using the caDSR Search Tree

The caDSR Search Tree lists the names of the different Contexts in caDSR and displays the various groupings of DEs within each Context. You can use the caDSR Contexts to help you quickly find forms, templates, and other collections of DEs.

 

 

 

 

More Information about the Groupings used in the caDSR Search Tree

  • Classifications and Classification Scheme Items. A Classification is simply a collection of related Classification Scheme Items, i.e., a mini-hierarchy within the tree. Data Elements, in turn, can be assigned to one or more Classification Scheme Items. The actual Classification assignments are managed by the Context administrator. Classification schemes that have workflow status of RELEASED are displayed in the tree.

Hovering the mouse over a classification scheme will display its definition; e.g.

  • Protocol Form Templates. Data Elements can be used on a form. For clinical trials, this will be a case report form or CRF. Many of these forms have been generalized into template forms that can be re-used. For the Cancer Therapy Evaluation Program Context (CTEP), these templates have been grouped by Phase and Disease. Within each of these groupings (Phase and Disease), they are further grouped by type of template. Currently, CTEP has templates for Phase III trials only. Clicking on one of these template form names in the tree will return the data elements for that form in the right pane.
  • Protocol Forms: These are forms (CRFs) that are used in a clinical trial. Note: Protocol forms that belong to the CTEP Context are not shown in the tree.
  • Specialized groupings. Other types of groupings can be found in the tree as well. For example, the Cancer Therapy Evaluation Program Context groups data elements based on CDE disease committee decisions into core and non-core categories that can be found in the tree, under each type of disease classification.
  • Classification Scheme Hierarchies. Classification Schemes can be defined in a hierarchy where one Classification Scheme can be the "parent" of other Classificatoin Schemes, and they in turn can be parents of their own "child" Classification Schemes (which would be the "grandchildren" of the original Classification Schemes). These structures are shown in the Navigator Tree.

Finding a Form using the caDSR Search Tree

To find forms and templates and their associated DEs in the CDE Browser, you need to use the caDSR Search Tree. (Note: Form Builder, a new module added to the CDE Browser in Release 2.1, provides a host of additional tools for finding, viewing, downloading, creating, editing, and maintaining forms and templates. For more information, read the Form Builder Help.)

There are two different ways to search for forms and templates using the caDSR Search Tree.

  • Searching in the Forms Catalogue: If you dont know to which context the form or template you want belongs or if you simply want to see a list of all the forms and templates that are available across all contexts, we recommend that you begin your search in the Forms Catalogue folder on the caDSR Search Tree. The Forms Catalogue is a central directory that lists all forms and templates that have been published in Form Builder. The Forms Catalogue folder is included as part of the caBIG Context. Forms and templates are listed in the here alphabetically and (if applicable) by protocol.

 

  • Searching in a Specific Context: If a form or template has not been added to the Forms Catalogue yet, you can search for it within a specific context. However, since each context organizes it forms using different classification schemes, you may need to explore the subfolders within each context before you find what you are looking for. But, in general, the method for finding a folder using this approach is the same. For example, to find the Advanced Bladder Cancer Follow-up protocol form template owned by the CTEP Context, you would simply follow the steps outlined below:

1.       Click the folder icon for Cancer Therapy Evaluation Program to expand the contents of that folder.

2.       Select the folder icon for Protocol Form Templates.

3.       Once the contents of the Protocol Form Templates folder are displayed, select the folder icon for Disease

4.       Select the Bladder folder icon.  Once the Bladder folder has been expanded, you will see all the template types (i.e. follow-up, on-study, etc.).

 

Note: The speed of your Internet connection will affect the response of the CDE Browser. If you are on dial-up, the wait could be significant.  Also, it is important to place the mouse pointer directly on the folder icon with the + sign.  If the pointer is on the Cancer Therapy Evaluation Program link, the CDE Browser will change the search parameters to the CTEP Context only, and will not display the sub-folders.).

5.       Select the type of template form you want to see. For this example, click the Follow-Up folder.

6.       Click the template form name link.  Once the link has been select, the search window on the right side of the screen will display the CDEs associated with that form.  The top of the screen will display the hierarchical level that you have currently selected. The search display at the bottom of the screen lists all the CDEs that are associated on the template. 

7.       If you want to see the electronic image of the template, select Download Template, located between the search buttons and the search result screen. There are a few templates that do not have the image file loaded, so if you select Download Template for these templates a message stating that no document is found will display.

Viewing Details about Object Classes and Associations

The Object Class Details page was added to the CDE Browser as part of Release 3.0.1. It provides users with a way (without leaving the CDE Browser) of viewing key information about the domain model to which a DE may be linked. For example, users can view details about the objects or classes in the domain as well as the associations that exist between the objects. Note: In this release, detailed information about objects and associations will be available for viewing in the Object Class Browser only for those DEs that were created in caDSR using the UML Loader.

 

 

Opening the Object Class Details Page:

 

  1. From the DE search results page, click on the DE you are interested in viewing.

The DE Details page will open.

  1. Click the DEC tab.
  2. Scroll down to the Object Class section on the DEC page, and then click the More Details link.

 

The Object Class Details page will open in a new window. There will be two tabs: Object Class and Associations.

 

 

 

 

Viewing Information about Objects

The Object Class page provides the following information about the objects in the underlying domain model.

         Concepts. All object classes in caDSR are linked to one or more concepts found in the EVS terminology server. This sections displays, for each concept, the Concept Name, Concept Code, Public ID, Definition Source, and EVS Source.

         Inheritance. If an object has an inheritance relationship to another object, this information will be displayed here.

  • Classification. When a UML model is loaded into caDSR using the UML Loader, the DEs and Object Classes that are created are automatically assigned classification schemes by the tool. This section will contain information about these classification scheme assignments.

         Alternate Names. When a UML model is loaded into caDSR using the UML Loader, the Object Classes that are created are automatically assigned Alternate Names based on information in the UML model. These Alternate Names are also automatically assigned to a set of classification schemes. This information will be displayed here.

  • Alternate Definitions. When a UML model is loaded into caDSR using the UML Loader, the Object Classes that are created are automatically assigned Alternate Definitions based on information in the UML model. These Alternate Definitions are also automatically assigned to a set of classification schemes. This information will be displayed here.

 

 

Viewing the Associations between Objects

 

The Associations tab lists all the outgoing, incoming, and bi-directional associations for a particular object. From this page, you can move around the domain model by navigating the associations. That is, you can shift the focus of your view from one object to another in the model.

 

 

 

 

 

Navigating the Object Model

The Object Class Details displays the details and associations for a given object. If you want to view the other objects and associations in the underlying domain model, you can do so by clicking the Navigate to this Association link. When you select this link, the page will refresh and show you the details and associations from the perspective of the target object. You can repeat this process until you have viewed all the objects in the model. At any point, if you want to return to the first object you viewed, just click the Clear Navigation Path link.

Using the CDE Compare Tool

The CDE Compare Tool allows you to view a selected set of Data Elements in a convenient, side-by-side format. To use the Compare Tool, simply follow the steps listed here:

1.       Run a search for CDEs from the Data Element Search page.

2.       Put a checkmark next to the CDEs you want to compare.

3.       Click the Add to CDE Compare List button. This will add the selected items to the list of CDEs to be compared.

4.       If you want to run another search and add more CDEs to the Compare list, repeat steps 1 through 3 above.

5.       When you are ready to view the CDEs youve selected, click the Compare CDEs button to open the Compare Tool.

 

The CDE Compare Tool displays most of the key details pertaining to each CDE. Use the hyperlinks at the top of the page (as shown below) to jump to the section that contains the details youre most interested in reviewing.

 

How to Change the Order of the CDEs in the Compare Tool

  1. Use the drop down list above the CDEs you want to move to specify the preferred order. Note: You do not need to specify a new location for each of the CDEs in the list. You can simply specify the CDEs you want to move.

  1. When you are ready to move the CDEs, click the Change Compare Order button.

Form Builder will insert the CDEs you have chosen to move into their new specified locations. The other CDEs in the table, if they are affected by this change, will be shifted to the right and placed in the first column not specifically reserved by you. For example, suppose the CDEs are currently arranged as follows: A, B, C, D, E. If you specify that you want to move D to the first spot, Form Builder will rearrange the CDEs into the following order: D, A, B, C, E. If, instead, you had specified that you wanted D in the first column and E in the second column, the new order would be as follows: D, E, A, B, C.

 

How to Remove CDEs from the Compare Tool

  1. Mark the checkbox next to the CDE you want to remove. If you want to remove all, click the Check All button.
  2. Click the Remove from Compare List button.

The CDEs you selected for removal will be removed from the Compare Tool.

 

How to Add more CDEs to the Compare Tool

You can add more DEs to the Compare Tool at any time. To do so, follow the steps listed below:

  1. Click the Done button to close the Compare Tool and return to the Data Element Search page.
  2. Search for the CDEs you want to add.
  3. Mark the checkbox next to the CDEs you want and then click Add to CDE Compare List.
  4. Click Compare CDEs to re-open the comparison table.

 

How to Export the CDEs in the Compare Tool to Excel

You can export the CDEs to Excel for offline sorting, manipulation, and analysis. The export will include all of the CDEs in the Compare Tool and will retain the same basic side-by-side layout.

  1. To download the CDE Comparison table to Excel, click the Excel download button.
  2. Click Save when prompted.

Exporting from the CDE Browser

The CDE Browser offers the following download options:

  • Download Search Results: You can download the results of your data element search to an Excel file or to an XML file. The download will include all the DEs in the search results table.
  • Download the contents of the CDE Cart: Use this option if you want to restrict the download to only those CDEs youve added to the CDE Cart. You can run several searches, add the DEs you want to the cart, and then download the DEs to an Excel file or XML file.
  • Download from the CDE Compare Tool: Export CDEs from the CDE Compare Tool to an Excel file. Retains the same side-by-side format. Allows further offline comparison.
  • Download an attached Form Template: If the appropriate file has been previously uploaded into the caDSR database and attached to the form, you can open the attached file from the CDE Browser and then save it to your local machine.

Note: Additional downloading options are available from the Form Builder.

 

Exporting the DE Search Results to an Excel or XML File

Click the Download Data Elements to Excel link to export all the data elements in the search results table to a comma-separated value Excel file. First, a notification window is displayed warning you that the downloading process will take a few seconds. Then, the File Download window is displayed with the Open or Save buttons. When the Open button is pressed, the Excel file is displayed in a separate window. When the Save button is pressed, the Save As window is displayed. You must manually close the notification window after the download process is completed.

Click the Download Data Elements as XML link to save all the data elements in the search results table to an XML file. First, the notification window is displayed warning you that the downloading process will take a few minutes. Please wait, as the XML file building process is considerably slower than that for Excel. Then the File Download window is displayed with the Open or Save options. We suggest selecting the Save option rather than Open, since directly opening the XML output appears to crash some browsers. After the file is saved you can view it in your favorite XML tool. You must manually close the notification window after the download process is complete.

Note: All the CDEs in the Search Results table will be downloaded. If you want to limit the CDEs that get exported or export a set of CDEs collected from several searches, you can download from the CDE Cart or CDE Compare Tool.

Understanding Form "Locking"

Editing a Form acquires a "lock" on all of the data related to that form modules, questions, valid values, etc. that prevents another user from making any changes to any of that information while the original user is working on the form. Clicking the [button name] saves the Form data and releases the lock.

While a form is locked other users can view the locked form and perform the following actions: Send a Sentinel Alert, Select Printer View, and View Reference Documents

While a form is locked other users can not Edit the form, Select for Copy , Assign Designations, Version the Form, Delete the Form, Perform an Excel Download of the Form, or Copy a module from the locked form to another form or onto the module cart.

Usage Notes:

         The lock on a Form will expire after 5 hours of inactivity on the part of the user session that obtained the lock. Users are encouraged to Logout from Form Builder when they have completed their edits.

         The lock is based on the user. A given user who connects to Form Builder through a second session will be able to perform any of the actions allowed during the initial session.

Notes:

1. The second session will not see any unsaved changes made during the first session.

2. Any unsaved changes from the first session that are saved after the second session has itself saved changes will overwrite those second session changes if they involve the same Form element (Question, Module, etc.).

Skip Patterns

A skip pattern defines a non-sequential path through a form. The skip pattern can adjust the questions asked a respondent based upon that respondents answers to preceding questions. The Form Builder user can define a skip pattern at the module level or at the valid value level for those Questions associated with enumerated Value Domains.

The following restrictions are placed on defining skip patterns:

  • The targets of skip patterns can only be protocols associated with the form from which the skip pattern originates.
  • A Skip Pattern can be associated only with Classifications already assigned to the form
  • A Module or Valid Value can be the origin of more than one skip pattern but the target location must be different for each.
  • Two Skip Patterns cannot target the same Protocol or Classification
  • A Module or Question that is the target of Skip Pattern cannot be deleted unless the Skip Pattern is updated to remove that Module/Question as being a target
  • You cannot delete a protocol-form association if the form is associated with a skip pattern.
  • You cannot delete a Classification associated with a Form if it is associated with a skip pattern. The skip Pattern needs to be updated before the Classification can be removed.

 

Adding a Skip Pattern (Valid Value)

    1. Select the Form that be the origin of the Skip Pattern and edit the Question that has the Valid Value from which the Skip Pattern will start
    2. Click the Add Skip Pattern link. The Form Builder will open the Select skip to location page and display the modules and questions of the same form
    3. Select the Skip to this module or Skip to this question link as appropriate to your Skip Pattern design. The Skip Pattern page will open.
    4. Review the Skip Pattern that you have defined. Check the Protocols and Classification Schemes to be included in the Skip Pattern and enter a Skip Pattern Instruction (optional).
    5. Click [Save] to record the Skip Pattern or [Cancel] to return to the module edit page without saving the Skip Pattern.

Copying a Module

You can reuse entire sets of questions by copying form modules from one or more source forms to a target form. Form Builder provides two techniques for copying modules:

  • Copying a single module from the source form to the target form; or
  • Copying multiple modules from one or more source forms to the module cart and then copying the modules from the module cart to the target form.

The module cart enables you to efficiently design a new form by reusing existing modules from multiple sources. You can then fine tune this design to complete the new form.

In the following instructions the target form is the form to receive copied modules; the source form is the form from which a module is copied.

Copying a single module:

1.       Select and edit the target form

2.       Determine the place in the form where the copied module is to be placed; click the Copy Module from Form link. This will open the Form Builder search page.

3.       Use the Form Builder search process to find the source form and edit that form

4.       Navigate to the top of the module you wish to copy. Click the link Copy this module to form at the top of the module definition. Form Builder will then take you back to your target form and copy the source module at the position you originally designated.

Select Copy module from module cart to copy the modules in the cart to the target form

 

Select Copy module from a form to start the module copy process
 

Select Copy this module to form to copy the source module to the target form
 

Copying modules using the Module Cart

This process reverses the order of the single module copy the source forms are visited first to select the source modules, and then the target form is edited to receive the modules copied to the module cart.

1.       For each form that will provide a module to be copied to the module cart

a.       View the details of the form and locate the source module

b.       Click the link Copy to module cart at the top of the source module to copy it to the module cart. Form Builder will display a message confirming the copy succeeded (Module(s) copied to module cart)

c.       Search for the next source form or edit the target form if you are done copying modules to the module cart

2.       Search for and edit the target form and navigate to the place where you want the modules on the module cart copied

3.       Click on the Copy module from module cart link to copy the module(s) from the module cart to the place youve indicated on the target form.

Working with Form Versions

Creating a new version of a form copies all of the source forms modules, questions, and related properties to a new copy of the form. This preserves the old form and its links within the caDSR and enables the user to make changes to the new version without invalidating these prior relationships. Once a form has multiple versions you can specify which version is the latest version. This is useful if a prior version is to be used while a new version is being finalized.

To create a new form version:

1.       Find the form via the Form Search page, and click on its name to view its details

2.       Click on the [Create New Version] button to open the Create New Version page, which will display the basic information about the form being versioned.

3.       Enter the version number of the new form. The new version number must be a number greater than the maximum version number displayed.

4.       Optionally, use the Change Note box to record information about the new version

5.       Check the Proceed to Edit the New Version checkbox if you want to work on the new form version after you save it.

6.       Click the [Save] button to save the new form version or click [Cancel] to exit the page without saving the new form version.

The new form version must be greater than the maximum version number recorded for the form
 

 

To specify a form version as the latest version:

  1. Find the form via the Form Search page, and click on its name to view its details
  2. Click the [Versions] button at the top of the form details page. This will display information on the various versions of the form
  3. Click the radio button adjacent to the form version that you want to specify as the latest version
  4. Click the [Save] button to make your selection permanent; click [Cancel] to return to the Form Details page without changing the latest version information.

Deleting a Form or Template

Deleting a form from Form Builder removes the form from the caDSR database. However, the CDEs associated with the form are not deleted.

 

To delete a form from the Form Search page:

  1. Go to the Form Search page and enter the appropriate search criteria.
  2. In the results table, click the Delete icon next to the form you want to delete.
  3. Click Yes to confirm the deletion. Form Builder will display a message indicating that the form has been successfully deleted. Once a form has been deleted, it cannot be retrieved. Deleting the form does not delete the associated CDEs from the caDSR database.

 

To delete a form from the Edit Form page:

  1. Open the form in Edit mode.
  2. Click Delete.
  3. Click Yes to confirm the deletion. The form will be permanently removed. The CDEs associated with the form will not be deleted from caDSR.

Printing a Form

For the best results when printing a form, we recommend using the Printer Friendly Version.

 

  1. Go to the Form Search page and find the form you want to print.
  2. In the search results table, click on the form name to open the form in the View Form mode.
  3. Click the Printer Friendly Version link. The form will open in a new window.
  4. From the Browsers File menu, select Print.

Publishing a Form to the Forms Catalogue

 

You can opt to publish any form that you own. When you publish a form, it gets added to the list of forms in the Forms Catalogue. The Forms Catalogue is a central directory where all published forms are listed. The Forms Catalogue appears in the caDSR Search Tree as part of the caBIG Context. Any CDE Browser or Form Builder user can view a form that is listed in the Forms Catalogue.

 

 

To publish a form:

  1. Open Form Builder and search for the form you want to publish.
  2. In the search results table, click on name of the form you want to publish so that the form opens in the View Form page.
  3. Click the Publish button at the top of the page. The form will be added to the Forms Catalogue, which can be accessed from the Search Tree.

Note: If you are logged into Form Builder with the Guest account, you will not have the option of publishing a form to the Forms Catalogue.

 

To remove a form from the Catalogue:

  1. Repeat steps 1 and 2 above.
  2. Click the Unpublish button.

Downloading Reference Documents

If a copy of the original paper version of the form or any other file has been previously uploaded into caDSR and attached to the form as a reference document, you can download it from Form Builder. Note: Files will be available for download in the format they were uploaded in.

To download a reference document:

  1. Go to the Form Search page and find the form you want to download.
  2. Click the Edit Form icon.
  3. Click the Reference Documents button.
  4. Click the name of the file you want to download.
  5. When prompted to Save or Open the file, click Save.

Exporting a Form to Excel

 

From Form Builder, you can export a form to Excel. The data elements and most of the form information (such as headers, footers, instructions, and grouping of questions into modules) will be included as part of the export.

 

To Export a Form to Excel:

  1. Go to the Form Search page and find the form you want to download.
  2. In the search results table, click on the form name to open the form in the View Form page.
  3. Click the Excel Download link.
  4. When prompted, click Save or Open. We recommend that you Save the file before opening it.

Using the CDE Cart

The CDE Cart functions in many ways like a typical shopping cart used at an online store.

 

Adding CDEs to the Cart

Data Elements can be moved to the CDE Cart from the Data Elements Search page. The elements in the Cart can then be downloaded in Excel or XML format or used with Form Builder to create or update forms and templates. The CDE Cart provides a convenient way for users to collect a variety of CDEs for downloading or viewing from a diverse set of searches.

1. From the Data Elements search results table, select the CDEs that you want to move to the Cart.
 

2. Click Add to CDE Cart. Note: This will not open the Cart. To view the contents of the Cart, click the CDE Cart icon at the top of the page.
 

 

Opening the Cart

To open the Cart, simply click the CDE Cart icon in the top right-hand menu.

 

Saving CDEs in Cart for Future Retrieval

The CDE Cart is available in secure and non-secure modes. When used in the secure mode as part of Form Builder, the items in the Cart can be saved to the database so that they remain in the Cart even after you have logged out and exited the Browser. When used in the non-secure mode, the items persist in the Cart only for the duration of the active session.

After adding CDEs to the Cart, you must also save them if you want them to be available during future sessions.

Exporting CDEs from the CDE Cart

The CDE Cart allows you to export collections of CDEs to an Excel or XML format.

  1. Add the CDEs that you want to export to the Cart. Note: All of the CDEs in the Cart will be included in the export.
  2. Click either the Download to Data Elements to Excel or the Download Data Elements as XML link.
  3. Follow the instructions that appear on the screen.

Note: We recommend that you choose the Save option instead of the Open option when prompted. Opening an XML file without first saving it to your local machine may cause problems in some cases.

Technical Tips

Standard Browser Navigation Buttons Not Supported

The browser's Back, Forward, and Refresh buttons are not fully supported. Use of these buttons could lead to unexpected results. Users are encouraged to use the navigation tools provided within CDE Browser and Form Builder.

Exporting Data

When exporting data from CDE Browser and Form Builder, you will be prompted to either Open or Save the resulting file. We recommend that you choose Save.

Recommended Settings and Supported Browsers

The application performs best when used as follows:

         Set the screen resolution to 1024x768

         Use either Netscape 6.x or Internet Explorer version 5.x or higher

         When using Form Builder, use Internet Explorer (v.6 or higher) or Mozilla Firefox. Form Builder may not be fully functional when used with Netscape Navigator.
